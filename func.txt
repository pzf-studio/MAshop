1. АДМИНИСТРАТИВНАЯ ПАНЕЛЬ (BACKOFFICE)

1.1. Сущности данных
В системе реализуются две основные сущности:
- **Раздел (Category)**
- **Товар (Product)**

1.2. Модель данных: Раздел (Category)
Атрибуты:
- `name`: String. Отображаемое название раздела.
- `index`: String|Integer. Уникальный идентификатор (индекс) для выборки данных из базы.

1.3. Модель данных: Товар (Product)
Атрибуты:
- `name`: String. Наименование товара.
- `article`: String. Уникальный артикул.
- `description`: String. Детальное описание.
- `advantages`: String. Список преимуществ в строго заданном формате "Текст:_Текст".
- `category`: Category. Ссылка на раздел, созданный в панели управления.
- `in_stock`: Boolean. Флаг наличия товара. Определяет видимость товара на сайте.
- `is_color_palette`: Boolean. Активирует функционал цветовой палитры.
- `palette_variants`: Integer [1-5]. Количество вариаций товара (N), актуально при `is_color_palette = true`. Для каждой вариации генерируется отдельная страница с артикулом в формате `АртикулCopyM`, где M ∈ [1, N].
- `images`: Array[File]. До 5 изображений. Реализовать адаптивную обработку изображений для корректного отображения в заданных рамках (масштабирование без выхода за границы контейнера).

1.4. Сохранение данных
При взаимодействии с кнопкой «Сохранить» в админ-панели, все введенные данные (разделы и товары) сериализуются и сохраняются в файл `DataBs.txt`.
*Примечание: Использование TXT-файла является временным решением с последующей миграцией на реляционную СУБД.*

2. КЛИЕНТСКАЯ ЧАСТЬ (FRONTEND)

2.1. Главная страница (`index.html`)
- Статичный контент.
- Модуль "Рекомендуемые товары":
    - Динамически отображает 3 случайных товара из базы данных.
    - При отсутствии товаров в базе отображается заглушка с текстом "Заглушка".

2.2. Страница каталога (`shop.html`)
- Динамически генерирует сетку товаров, подгружая данные из базы.
- При отсутствии товаров отображается страница технического обслуживания (стили берутся из `shop.css`).

2.3. Функционал корзины
- Реализована в виде модального (всплывающего) окна, вызываемого с правой стороны экрана.
- **Состав данных в корзине:**
    - Список позиций: наименование, количество, цена за единицу.
    - Итоговая сводка: общее количество товаров и общая стоимость.
- **Сохранение состояния:** Данные корзины (наименование, артикул, цена, количество) должны сохраняться в `localStorage` браузера для обеспечения персистентности между сессиями и переходами по страницам.

2.4. Детальная страница товара (`piece.html`)
- URL формируется по шаблону: `/shop/<Артикул>`.
- Данные на странице подгружаются динамически на основе артикула, переданного в URL.
- Функционал:
    1. Добавление товара в корзину.
    2. Режим просмотра изображения в полноэкранном режиме (lightbox).
- Стилизация определяется файлом `piece.css`.

3. ОФОРМЛЕНИЕ ЗАКАЗА

3.1. Страница оформления заказа (`checkout.html`)
- На страницу передается и отображается текущее состояние корзины пользователя.
- Предоставляется форма для ввода данных клиента.

3.2. Процесс оформления
После заполнения формы и нажатия кнопки «Оформить заказ»:
- Происходит валидация данных формы.
- Данные заказа (состав корзины + информация о клиенте) передаются методом POST на серверный обработчик.
- Серверный обработчик формирует и отправляет уведомление с деталями заказа в заданный Telegram-чат через API Telegram Bot для дальнейшей обработки менеджером.
